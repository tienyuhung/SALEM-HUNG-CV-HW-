# Photoshop 压缩图片指南

## ✅ 可以使用 Photoshop 压缩图片

使用 Photoshop 压缩图片是一个很好的解决方案，可以：
- ✅ 减小文件大小，便于上传到 GitHub
- ✅ 提高网站加载速度
- ✅ 保持图片质量
- ✅ 节省存储空间

---

## 📐 推荐的压缩参数

### 对于 Web 使用的图片

#### 1. **首页背景图** (`hero-image.jpg`)
- **尺寸**：建议宽度 1920px（保持比例）
- **质量**：80-85%
- **格式**：JPEG
- **目标大小**：1-2MB

#### 2. **About 头像** (`salemhung2025-3.jpg`)
- **尺寸**：建议宽度 800-1000px（保持比例）
- **质量**：85-90%
- **格式**：JPEG
- **目标大小**：500KB-1MB

#### 3. **Portfolio 作品图**
- **尺寸**：建议宽度 1200-1600px（保持比例）
- **质量**：80-85%
- **格式**：JPEG
- **目标大小**：500KB-1.5MB

## 🎨 Photoshop 压缩步骤

### 方法一：使用"存储为 Web 所用格式"（推荐）

1. **打开图片**
   - 在 Photoshop 中打开要压缩的图片

2. **选择"存储为 Web 所用格式"**
   - 菜单：`文件` → `导出` → `存储为 Web 所用格式（旧版）`
   - 或使用快捷键：`Alt + Shift + Cmd + S` (Mac) / `Alt + Shift + Ctrl + S` (Windows)

3. **设置压缩参数**
   - **预设**：选择 "JPEG 高" 或 "JPEG 中"
   - **质量**：调整滑块到 80-85%
   - **优化**：勾选 "优化"
   - **格式**：选择 JPEG
   - **品质**：80-85（数值越高，质量越好，文件越大）

4. **预览和调整**
   - 在预览窗口中查看效果
   - 调整质量滑块，找到质量和文件大小的平衡点
   - 左下角会显示文件大小

5. **保存**
   - 点击 "存储"
   - **重要**：保存时使用**新文件名**（如 `hero-image-compressed.jpg`）
   - 确认压缩效果后再替换原文件

### 方法二：使用"导出为"（Photoshop CC 2018+）

1. **打开图片**
   - 在 Photoshop 中打开要压缩的图片

2. **选择"导出为"**
   - 菜单：`文件` → `导出` → `导出为`
   - 或使用快捷键：`Shift + Cmd + Alt + W` (Mac) / `Shift + Ctrl + Alt + W` (Windows)

3. **设置参数**
   - **格式**：选择 JPEG
   - **品质**：调整到 80-85%
   - **宽度/高度**：可以调整尺寸（保持比例）
   - **优化**：勾选

4. **导出**
   - 点击 "导出"
   - 保存为新文件

### 方法三：使用"图像大小"调整尺寸

如果图片尺寸过大，可以先缩小尺寸：

1. **打开图片**
2. **调整图像大小**
   - 菜单：`图像` → `图像大小`
   - 或快捷键：`Cmd + Option + I` (Mac) / `Ctrl + Alt + I` (Windows)

3. **设置尺寸**
   - **宽度**：设置为合适尺寸（如 1920px 用于背景图）
   - **约束比例**：确保勾选（保持宽高比）
   - **分辨率**：72 DPI（Web 使用）

4. **应用**
   - 点击 "确定"
   - 然后使用上述方法压缩质量

## ⚠️ 重要提示

### 1. **备份原文件**
- 压缩前，建议先备份原文件
- 或者使用新文件名保存，确认效果后再替换

### 2. **保持图片质量**
- 质量不要低于 75%，否则可能影响视觉效果
- 在预览窗口中检查压缩后的效果
- 确保文字和细节仍然清晰

### 3. **保持宽高比**
- 压缩时务必保持原始宽高比
- 不要拉伸或变形图片

### 4. **测试压缩效果**
- 压缩后，在浏览器中打开网站测试
- 确保图片显示正常
- 检查加载速度是否改善

## 🔄 批量压缩（如果有多个文件）

### 使用 Photoshop 动作（Action）

1. **录制动作**
   - 窗口 → 动作
   - 创建新动作
   - 录制压缩步骤

2. **批处理**
   - 文件 → 自动 → 批处理
   - 选择录制的动作
   - 选择源文件夹和目标文件夹
   - 执行批处理

### 或使用其他工具

- **ImageOptim**（Mac）：https://imageoptim.com
- **TinyPNG**（在线）：https://tinypng.com
- **Squoosh**（在线）：https://squoosh.app

## 📝 压缩后替换文件

压缩完成后：

1. **测试压缩后的图片**
   - 在浏览器中打开 `index.html`
   - 检查图片显示是否正常

2. **替换原文件**
   - 如果效果满意，用压缩后的文件替换原文件
   - 保持相同的文件名

3. **上传到 GitHub**
   - 使用 Git 命令行上传
   - 或使用 GitHub Desktop
   - 压缩后的文件应该可以正常上传

## 💡 额外建议

### 如果图片仍然太大

1. **进一步降低质量**：尝试 75-80%
2. **缩小尺寸**：如果不需要那么大的尺寸
3. **使用 WebP 格式**：更小的文件大小（需要代码支持）

### 优化建议

- **首页背景图**：可以稍微降低质量，因为会转为黑白
- **Portfolio 图片**：保持较高质量，因为需要展示细节
- **头像**：保持高质量，因为是个人形象
