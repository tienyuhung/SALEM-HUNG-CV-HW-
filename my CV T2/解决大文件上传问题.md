# 解决 GitHub 大文件上传问题

## 🔍 问题原因

GitHub 网页界面上传文件时，如果单个文件超过 **50MB** 会有问题。虽然您的文件没有超过 50MB，但某些大文件（如 7MB 的图片）可能仍会遇到上传限制。

**检测到的大文件**：
- `hero-image.jpg`: 7.1MB
- `salemhung2025-2.jpg`: 7.1MB
- `portfolio2-3.jpg`: 3.9MB
- `portfolio2-4.jpg`: 3.6MB
- `portfolio8.jpg`: 3.4MB

---

## ✅ 解决方案

### 方案一：使用 Git 命令行上传（推荐）

Git 命令行可以处理更大的文件，是最可靠的解决方案。

#### 步骤 1：安装 Git（如果还没有）

**Mac 用户**：
```bash
# 检查是否已安装
git --version

# 如果没有安装，使用 Homebrew
brew install git
```

#### 步骤 2：初始化 Git 仓库

在终端中执行：

```bash
# 进入项目文件夹
cd "/Users/salemhung/Library/Mobile Documents/com~apple~CloudDocs/Desktop/台科碩班研究所/Cursor HW/my CV T2"

# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交文件
git commit -m "Initial commit - Personal portfolio website"

# 设置主分支
git branch -M main
```

#### 步骤 3：连接到 GitHub 仓库

```bash
# 添加远程仓库（替换为您的实际仓库地址）
git remote add origin https://github.com/您的用户名/您的仓库名.git

# 推送到 GitHub
git push -u origin main
```

**示例**：
如果您的用户名是 `salemhung`，仓库名是 `my-cv-t2`：
```bash
git remote add origin https://github.com/salemhung/my-cv-t2.git
git push -u origin main
```

#### 如果遇到认证问题

GitHub 现在需要使用 Personal Access Token 而不是密码：

1. **生成 Token**：
   - 访问 https://github.com/settings/tokens
   - 点击 "Generate new token" → "Generate new token (classic)"
   - 勾选 `repo` 权限
   - 点击 "Generate token"
   - **复制并保存 Token**（只显示一次）

2. **使用 Token**：
   ```bash
   # 推送时会要求输入用户名和密码
   # 用户名：您的 GitHub 用户名
   # 密码：使用刚才生成的 Token（不是 GitHub 密码）
   git push -u origin main
   ```

### 方案二：压缩图片文件（推荐用于优化）

压缩图片可以减小文件大小，提高网站加载速度。

#### 使用在线工具压缩

1. **TinyPNG**（推荐）：
   - 访问 https://tinypng.com
   - 上传图片文件
   - 下载压缩后的图片
   - 替换原文件

2. **Squoosh**：
   - 访问 https://squoosh.app
   - 上传图片并调整质量
   - 下载压缩后的图片

#### 使用命令行工具（Mac）

```bash
# 安装 ImageMagick（如果还没有）
brew install imagemagick

# 压缩单个图片（保持质量 85%）
convert hero-image.jpg -quality 85 hero-image-compressed.jpg

# 批量压缩所有 JPG 文件
for file in *.jpg; do
    convert "$file" -quality 85 "compressed_$file"
done
```

#### 压缩建议

- **hero-image.jpg** (7.1MB) → 压缩到 1-2MB
- **salemhung2025-2.jpg** (7.1MB) → 压缩到 1-2MB
- **Portfolio 图片** → 压缩到 500KB-1MB 即可

**注意**：压缩后记得测试图片质量是否满足需求。

### 方案三：使用 GitHub Desktop（图形界面）

GitHub Desktop 可以处理更大的文件，比网页界面更可靠。

#### 步骤 1：下载 GitHub Desktop

- 访问 https://desktop.github.com
- 下载并安装 GitHub Desktop

#### 步骤 2：登录并克隆仓库

1. 打开 GitHub Desktop
2. 登录您的 GitHub 账号
3. 点击 "File" → "Add Local Repository"
4. 选择项目文件夹
5. 点击 "Publish repository" 发布到 GitHub

#### 步骤 3：推送文件

1. 在 GitHub Desktop 中，所有文件会显示为 "Uncommitted changes"
2. 在左下角填写 Commit message
3. 点击 "Commit to main"
4. 点击 "Push origin" 推送到 GitHub

### 方案四：分批上传（临时方案）

如果必须使用网页界面，可以分批上传：

1. **第一批**：上传小文件（HTML、CSS、JS、小图片）
2. **第二批**：使用 Git 命令行上传大文件
3. **或者**：先压缩大文件，再上传

## ⚠️ 重要提示

1. **GitHub 文件大小限制**：
   - 单个文件：最大 100MB
   - 仓库总大小：建议不超过 1GB
   - 网页界面上传：实际限制可能更小（约 50MB）

2. **如果文件真的超过 100MB**：
   - 需要使用 Git LFS（Large File Storage）
   - 或者考虑使用其他存储方案（如 CDN）

3. **图片优化建议**：
   - Web 使用：JPG 质量 80-85% 即可
   - PNG：使用 TinyPNG 压缩
   - 考虑使用 WebP 格式（更小的文件大小）

## 🔧 故障排除

### 问题 1：Git push 时提示认证失败

**解决**：
- 使用 Personal Access Token 而不是密码
- 或者使用 SSH 密钥

### 问题 2：仍然提示文件太大

**解决**：
- 检查是否有文件超过 100MB
- 使用 `git lfs` 处理大文件
- 或者压缩文件

### 问题 3：上传很慢

**解决**：
- 大文件上传需要时间，请耐心等待
- 考虑先压缩图片再上传
