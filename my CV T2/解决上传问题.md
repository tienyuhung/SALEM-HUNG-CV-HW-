# 解决 GitHub 上传失败问题

## 如果网页界面上传失败，使用 Git 命令行是最可靠的解决方案

---

## 方案一：使用 Git 命令行（最可靠）⭐

### 完整步骤：

1. **打开终端（Terminal）**
   - 按 `Cmd + Space`，输入 "Terminal"

2. **进入项目文件夹**
   ```bash
   cd "/Users/salemhung/Library/Mobile Documents/com~apple~CloudDocs/Desktop/台科碩班研究所/Cursor HW/my CV T2"
   ```

3. **检查是否已有 Git 仓库**
   ```bash
   ls -la | grep .git
   ```
   - 如果显示 `.git`，说明已经初始化
   - 如果没有，需要初始化

4. **初始化 Git（如果还没有）**
   ```bash
   git init
   ```

5. **添加所有文件**
   ```bash
   git add .
   ```

6. **提交文件**
   ```bash
   git commit -m "Initial commit - Personal portfolio website"
   ```

7. **设置主分支**
   ```bash
   git branch -M main
   ```

8. **连接 GitHub 仓库**
   ```bash
   git remote add origin https://github.com/您的用户名/您的仓库名.git
   ```
   
   如果提示 "remote origin already exists"，先删除再添加：
   ```bash
   git remote remove origin
   git remote add origin https://github.com/您的用户名/您的仓库名.git
   ```

9. **上传文件**
   ```bash
   git push -u origin main
   ```

10. **输入凭证**
    - 用户名：您的 GitHub 用户名
    - 密码：使用 Personal Access Token（不是 GitHub 密码）

## 方案二：使用 GitHub Desktop（图形界面，更简单）

### 如果命令行不熟悉，可以使用 GitHub Desktop：

1. **下载 GitHub Desktop**
   - 访问：https://desktop.github.com
   - 下载并安装

2. **登录 GitHub 账号**
   - 打开 GitHub Desktop
   - 登录您的 GitHub 账号

3. **添加本地仓库**
   - 点击 "File" → "Add Local Repository"
   - 选择项目文件夹：`/Users/salemhung/Library/Mobile Documents/com~apple~CloudDocs/Desktop/台科碩班研究所/Cursor HW/my CV T2`
   - 点击 "Add Repository"

4. **发布到 GitHub**
   - 点击 "Publish repository"
   - 输入仓库名称
   - 选择 "Public"
   - 点击 "Publish Repository"

5. **等待上传完成**
   - GitHub Desktop 会自动处理大文件上传

## 方案四：分批上传文件

### 如果必须使用 GitHub 网页界面：

1. **先上传小文件**
   - 先上传 `index.html`、`styles.css`、`script.js` 等小文件

2. **使用 Git 命令行上传大文件**
   - 对于大图片文件，使用 Git 命令行单独上传

3. **或者压缩所有大文件**
   - 将所有超过 2MB 的文件都压缩

## 创建 Personal Access Token（如果 Git 要求输入密码）

1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 设置：
   - **Note**: 输入描述（如 "My CV Website"）
   - **Expiration**: 选择过期时间
   - **Select scopes**: 勾选 `repo`（完整仓库访问权限）
4. 点击 "Generate token"
5. **复制 Token**（只显示一次，请保存）
6. 当 Git 要求输入密码时，粘贴这个 Token

## 常见错误及解决方法

### 错误 1: "fatal: not a git repository"
**解决**：确保在项目文件夹中执行 `git init`

### 错误 2: "remote origin already exists"
**解决**：
```bash
git remote remove origin
git remote add origin https://github.com/您的用户名/您的仓库名.git
```

### 错误 3: "Authentication failed"
**解决**：
- 使用 Personal Access Token 而不是密码
- 检查 Token 是否过期
- 重新创建 Token

### 错误 4: "failed to push some refs"
**解决**：
```bash
git pull origin main --allow-unrelated-histories
git push -u origin main
```

### 错误 5: 上传很慢或中断
**解决**：
- 这是正常的，大文件需要时间
- 如果中断，重新执行 `git push` 即可
- 或者使用 GitHub Desktop
