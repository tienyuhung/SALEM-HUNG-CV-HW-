# 个人网站部署说明

## 部署方式选择

您的网站是纯前端静态网站（HTML、CSS、JavaScript），可以通过以下几种方式部署并分享链接：

---

## 方法一：Netlify（推荐，最简单）⭐

### 优点：
- 免费且快速
- 拖放即可部署
- 自动生成 HTTPS 链接
- 支持自定义域名

### 步骤：
1. 访问 [https://www.netlify.com](https://www.netlify.com)
2. 注册/登录账号（可使用 GitHub 账号快速登录）
3. 在 Netlify 控制台，找到 "Add new site" → "Deploy manually"
4. 将整个项目文件夹（`my CV T2`）拖放到上传区域
5. 等待部署完成（通常几秒钟）
6. 部署完成后，Netlify 会自动生成一个链接，例如：`https://random-name-123456.netlify.app`
7. 您可以：
   - 点击 "Site settings" → "Change site name" 修改为自定义名称
   - 分享这个链接给其他人

---

## 方法二：Vercel（推荐，也很简单）⭐

### 优点：
- 免费且快速
- 界面简洁
- 自动生成 HTTPS 链接

### 步骤：
1. 访问 [https://vercel.com](https://vercel.com)
2. 注册/登录账号（可使用 GitHub 账号快速登录）
3. 点击 "Add New" → "Project"
4. 如果代码在 GitHub，可以直接导入；如果没有，选择 "Deploy" → 上传文件夹
5. 将整个项目文件夹上传
6. 等待部署完成
7. 获得链接，例如：`https://your-project.vercel.app`

---

## 方法三：GitHub Pages（需要 GitHub 账号）

### 优点：
- 完全免费
- 与 GitHub 集成
- 链接格式：`https://yourusername.github.io/repository-name`

### 步骤：
1. 在 GitHub 创建新仓库（repository）
2. 将项目文件上传到仓库
3. 在仓库设置中，找到 "Pages" 选项
4. 选择 "Deploy from a branch"，选择 `main` 分支，文件夹选择 `/ (root)`
5. 保存后，GitHub 会生成链接：`https://yourusername.github.io/repository-name`
6. 等待几分钟后，网站即可访问

---

## 方法四：其他免费托管服务

### Surge.sh
- 网址：https://surge.sh
- 需要安装 Node.js
- 命令行部署，简单快速

### Firebase Hosting
- 网址：https://firebase.google.com
- Google 提供，稳定可靠

---

## 快速部署建议

**如果您想最快部署：**
1. 使用 **Netlify** 或 **Vercel**
2. 直接拖放文件夹即可
3. 几秒钟后获得链接

**如果您已有 GitHub 账号：**
1. 使用 **GitHub Pages**
2. 上传代码到 GitHub
3. 在设置中启用 Pages

---

## 注意事项

1. **确保所有图片文件都在项目中**：检查所有 `portfolio*.jpg`、`hero-image.jpg`、`salemhung2025-3.jpg` 等文件都已包含
2. **测试链接**：部署后，在不同设备和浏览器中测试网站
3. **自定义域名**（可选）：如果需要，可以在 Netlify/Vercel 设置中添加自己的域名

---

## 当前项目文件清单

确保以下文件都已包含在部署中：
- ✅ `index.html` - 主页面
- ✅ `styles.css` - 样式文件
- ✅ `script.js` - JavaScript 文件
- ✅ `hero-image.jpg` - 首页背景图
- ✅ `salemhung2025-3.jpg` - About 页面头像
- ✅ 所有 `portfolio*.jpg`、`portfolio*.webp`、`portfolio*.png` 文件
- ✅ 其他图片文件

## 需要帮助？

如果在部署过程中遇到问题，可以：
1. 检查浏览器控制台是否有错误（按 F12）
2. 确认所有文件路径正确
3. 确认图片文件都已上传
