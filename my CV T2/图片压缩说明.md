# 图片压缩说明

## 为什么压缩图片可以解决问题？

1. **GitHub 上传限制**：
   - 虽然单个文件限制是 100MB，但大文件上传容易失败
   - 5MB 的图片虽然不算太大，但压缩后可以：
     - 加快上传速度
     - 减少上传失败的风险
     - 提高网站加载速度

2. **网站性能**：
   - 压缩后的图片加载更快
   - 用户体验更好
   - 节省带宽

---

## 图片压缩方法

### 方法一：使用在线工具（最简单）⭐

#### 推荐工具：

1. **TinyPNG**（推荐）
   - 网址：https://tinypng.com
   - 支持 JPG 和 PNG
   - 可以压缩 70-80% 大小，质量损失很小
   - 免费，无需注册

2. **Squoosh**（Google 开发）
   - 网址：https://squoosh.app
   - 可以实时预览压缩效果
   - 可以调整压缩质量
   - 完全免费

3. **Compressor.io**
   - 网址：https://compressor.io
   - 支持多种格式
   - 简单易用

#### 使用步骤（以 TinyPNG 为例）：

1. 访问 https://tinypng.com
2. 拖放 `hero-image.jpg` 和 `salemhung2025-2.jpg` 到网页
3. 等待压缩完成
4. 下载压缩后的图片
5. 替换原文件

---

### 方法二：使用 Mac 内置工具

#### 使用"预览"（Preview）应用：

1. **打开图片**
   - 双击 `hero-image.jpg` 在预览中打开

2. **导出并调整质量**
   - 点击菜单 "文件" → "导出..."
   - 在格式中选择 "JPEG"
   - 调整 "质量" 滑块（建议 70-80%）
   - 点击 "存储"
   - 保存为新文件（例如：`hero-image-compressed.jpg`）

3. **检查文件大小**
   - 右键点击文件 → "显示简介"
   - 查看文件大小

4. **替换原文件**
   - 如果大小合适，替换原文件

---

### 方法三：使用命令行工具（适合熟悉命令行的用户）

#### 使用 ImageMagick（需要先安装）：

1. **安装 ImageMagick**
   ```bash
   brew install imagemagick
   ```

2. **压缩图片**
   ```bash
   # 压缩 hero-image.jpg（质量 80%）
   convert hero-image.jpg -quality 80 hero-image-compressed.jpg
   
   # 压缩 salemhung2025-2.jpg（质量 80%）
   convert salemhung2025-2.jpg -quality 80 salemhung2025-2-compressed.jpg
   ```

3. **检查文件大小**
   ```bash
   ls -lh hero-image-compressed.jpg salemhung2025-2-compressed.jpg
   ```

4. **替换原文件**
   - 如果大小合适，替换原文件

---

## 压缩目标

- **hero-image.jpg**: 从 5.0MB 压缩到 **2MB 左右**
- **salemhung2025-2.jpg**: 从 5.0MB 压缩到 **2MB 左右**

### 质量建议：
- **JPEG 质量**: 70-85% 通常可以获得很好的压缩比，同时保持可接受的质量
- **对于网站使用**: 80% 质量通常足够，肉眼几乎看不出差别

---

## 压缩后操作

1. **备份原文件**（可选但推荐）
   ```bash
   # 在项目文件夹中创建备份
   cp hero-image.jpg hero-image-backup.jpg
   cp salemhung2025-2.jpg salemhung2025-2-backup.jpg
   ```

2. **替换原文件**
   - 将压缩后的文件重命名为原文件名
   - 或者直接替换

3. **测试网站**
   - 在浏览器中打开 `index.html`
   - 检查图片显示是否正常
   - 确认质量可以接受

4. **上传到 GitHub**
   - 使用 Git 命令行上传（参考 `Git命令行上传步骤.md`）
   - 或者如果文件已压缩，可以尝试网页界面上传

1. 访问 https://tinypng.com
2. 拖放两张图片到网页
3. 等待压缩完成
4. 下载压缩后的图片
5. 替换项目中的原文件
6. 检查文件大小是否达到目标（约 2MB）
7. 测试网站显示效果
8. 上传到 GitHub

---

## 注意事项

1. **保留原文件备份**：压缩前建议备份原文件
2. **检查图片质量**：压缩后检查图片在网站上的显示效果
3. **文件命名**：确保压缩后的文件名与原文件名一致
4. **测试**：压缩后先在本地测试网站，确认一切正常

---

## 预期效果

压缩后：
- ✅ 文件大小减少 60-70%
- ✅ 上传到 GitHub 更容易成功
- ✅ 网站加载速度更快
- ✅ 用户体验更好
- ✅ 图片质量仍然可以接受（如果使用 70-80% 质量）

---

## 如果压缩后仍然太大

如果压缩到 2MB 后仍然无法上传：
1. 可以进一步压缩到 1.5MB
2. 或者使用 Git 命令行上传（不受大小限制）
3. 参考 `Git命令行上传步骤.md`
